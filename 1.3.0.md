## 需求：Agent 软件安装知识库系统

### 目标
为 SSH 操作 Agent 增加一个本地知识库模块，解决大模型不认识特殊软件（如 openclaw）而导致用错误包管理器搜索的问题。知识库需支持成千上万条软件安装/更新/卸载说明，并实现**只更新数据、不升级代码**的长期维护。

### 核心能力
1. 使用 Sqlite FTS5 全文搜索引擎存储和检索知识条目。
2. 应用启动时从远程（如 GitHub Releases 固定链接）下载最新 `knowledge.db` 文件，若下载失败则使用内置版本，保证离线可用。
3. 用户输入后，提取操作类型（安装/卸载/更新），执行 FTS5 全文搜索，取最相关条目。
4. 将命中的知识条目作为强制上下文注入到 LLM 请求中，确保 LLM 严格按命令执行，不再自行猜测包管理器。
5. 设计一套清晰的优先级与兜底规则，覆盖知识库命中/未命中、LLM 自身已知/未知的多种场景，杜绝危险搜索行为。

### 数据库结构（文件格式：Sqlite）
创建 `software_guides` 实体表与 `guides_fts` FTS5 索引表，字段包括：
- `id`：主键
- `software_name`：规范名称
- `aliases`：逗号分隔别名
- `op_type`：操作类型（install, uninstall, update, configure）
- `platform`：适用系统（linux, macos, windows, all）
- `package_manager`：推荐包管理器（npm, pip, apt 等）
- `summary`：一句话说明
- `steps`：详细步骤描述（给 LLM 参考）
- `commands`：可直接执行的命令，换行分隔
- `pre_requirements`：前置依赖说明
- `notes`：备注
- `mode`：策略模式（`strict` 强制使用此命令；`suggest` 仅推荐，允许优化参数但不可更换包管理器）

### 远程同步与更新机制
- 维护一个外部 CSV/Excel 文件用于编辑知识条目。
- 提供简单的转换脚本（Python 等），读取源数据 → 写入 Sqlite → 创建 FTS5 索引 → 输出 `knowledge.db`。
- 将 `knowledge.db` 发布至可直链下载的静态存储（如 GitHub Releases）。
- Flutter 端启动时，检查远程文件 ETag 或修改时间，若更新则下载覆盖本地数据库文件，失败则回退旧版。

### 检索与 Prompt 注入逻辑
1. 从用户输入中提取操作类型（简单关键词匹配：安装/卸载/更新等）。
2. 用原始输入作为 FTS5 查询条件，可附加 `op_type` 过滤，按相关性排序取 top 1（或 top 3 备选）。
3. 若命中（score > 阈值）：
    - 将命中条目的 `software_name`、`op_type`、`commands`、`pre_requirements` 等构造成极简指令，注入 LLM 的 system message 或 prompt 中。
    - 注入指令模板：“本地已验证知识库要求使用以下命令执行 {操作类型} {软件名}，你必须严格照做，不得修改、不得使用其他包管理器：[命令列表]”。
    - 若条目 `mode=suggest`，则指令为：“官方推荐方法如下，你可以调整非包管理器部分的参数，但不得更换包管理器：…”
4. 若未命中：不注入知识库，进入 LLM 自由模式（但受安全规则约束）。

### LLM 安全规则（固定在 System Prompt 中）
必须包含以下不可违背的规则：
- 当系统提供本地知识库命令时，必须原样使用，不得替换包管理器或更改命令核心逻辑。
- 当本地知识库未提供而 LLM 认识该软件时，正常使用正确的包管理器直接安装，**禁止使用任何搜索类命令**（如 `apt search`、`yum search`、`dnf search`）。
- 当 LLM 无法确定正确安装方式时，必须拒绝执行，并回复：“该软件暂无已知安装方式，请提供官方安装指引。”

### 优先级与兜底一览
| 场景 | 行为 |
|------|------|
| 知识库命中 + strict | 注入强制命令，LLM 仅传达并执行 |
| 知识库命中 + suggest | 注入推荐命令，LLM 可优化参数但不可换包管理器 |
| 知识库未命中，LLM 知道 | LLM 正常生成命令，但系统拦截搜索命令以防猜测 |
| 知识库未命中，LLM 不知道 | LLM 必须拒绝，提示提供安装方法 |
| 命令执行失败 | 捕获错误返回给用户，不进入 LLM 自救循环 |

### 非功能性要求
- 本地检索响应必须 < 50ms，条目数 10,000+ 时仍保持该性能。
- 完全离线可用，网络仅用于更新数据库文件。
- Flutter 端可使用 `sqflite`（移动端）/ `sqflite_common_ffi`（桌面端）实现。
- 代码实现后，新增知识只需更新数据库文件，无需修改 Flutter 代码或发布新版本。
