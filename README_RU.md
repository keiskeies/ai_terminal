## Одно предложение, чтобы объяснить это

> **Никогда не пользовались терминалом? Не проблема.** Откройте AI Terminal, скажите, что вы хотите, простым языком — он подключится к вашему серверу, выполнит команды, установит программное обеспечение и устранит неполадки. Всё безопасно и под вашим контролем.

## 🎯 Знакомо?

### 😫 Начинающие / Нетехнические пользователи

- Вы арендовали VPS, открыли терминал и уставились на **чёрный экран**, не понимая, что вводить
- Друг сказал «просто установи Nginx» — вы погуглили 10 руководств, в каждом разные команды
- Пытались настроить Java, неправильно отредактировали `PATH` и сломали весь терминал
- Кто-то предупредил об уязвимости сервера — вы даже не знаете, как проверить
- После 3 часов возни ничего не работает. Вы сдались.

### 👨‍💻 Разработчики

- Вы гуглите одни и те же команды `chmod` / `systemctl` каждый раз
- Подключаетесь по SSH к серверу и забываете точные флаги `grep`, которые нужны
- Хотите посмотреть логи? Сначала найдите ту закладку шестимесячной давности
- 15 открытых вкладок браузера, переключение между серверами, теряетесь, что где

### 🔧 DevOps / Системные администраторы

- Одно и то же ПО на 10 серверах? Подключайтесь по SSH к каждому и повторяйте. Снова.
- «Кто изменил этот конфиг?» — никто не помнит, ничего не логируется
- Новый сотрудник спрашивает «как настроить среду?» — вы объясняли это уже 5 раз в этом месяце
- Хотите сделать пакетную проверку работоспособности? Написание скрипта занимает больше времени, чем ручное выполнение

### 🧑‍💼 Продакт-менеджеры / Основатели стартапов

- Ваш единственный разработчик ушёл. Сервер теперь чёрный ящик.
- Нужно проверить какие-то данные, но вы не умеете писать SQL. Приходится кого-то спрашивать.
- Развёртывание изменения конфига требует спринта разработки. Это буквально одна строка.
- На вас 5 должностей. У вас нет времени учить `vi`.

**Все сценарии выше? Одно предложение AI Terminal решает их.**

## 💡 Что это может сделать для вас?

### Установить ПО? Просто скажите.

> 💬 «Установи Docker на этом сервере»

ИИ определяет версию вашей ОС, сопоставляет с официальной документацией, запускает команды установки и проверяет, что всё сработало. Никаких команд для запоминания.

### Настроить среды? Больше никаких головных болей с PATH.

> 💬 «Настрой Python 3.12 с правильными переменными окружения»

ИИ знает, что Debian использует `apt`, CentOS — `yum`, macOS — `brew`. Он не гадалок — он строго следует официальной документации.

### Проверить уязвимости? Он параноичнее вас.

> 💬 «Просканируй мой сервер на наличие проблем безопасности»

ИИ автоматически запускает проверки обновлений системы, сканирование портов и аудит процессов. Вы получаете полный отчёт о том, что нужно исправить.

### Читать логи? Больше никаких рысканий по закладкам.

> 💬 «Покажи последние ошибки Nginx»

ИИ знает, где хранятся логи, как их фильтровать и что важно. Ключевая информация, без гимнастики с `tail -f`.

### Управлять серверами? Несколько машин — один интерфейс.

Удалённые подключения SSH с пулом соединений. Переключение между серверами без задержек. Несколько вкладок — одно общее соединение.

## 🛡️ Безопасность: слон в комнате

Отдать свой сервер ИИ звучит пугающе. Три обоснованных опасения:

### 🔐 «Куда деваются мои пароли?»

```
Ваш пароль → Безопасное хранилище системного уровня (macOS Keychain / Android Keystore)
                       ↓
              Локальная база данных хранит только «какой ключ использован», никогда не сам пароль
                       ↓
              Пароли никогда не появляются в открытом виде в логах, конфигах или на диске
```

Даже если кто-то украдёт ваше устройство, без биометрии/код-пароля он получит только зашифрованную абракадабру.

### 🤖 «Может ли ИИ выйти из-под контроля?»

**Нет.** Три уровня защиты:

```
┌─────────────────────────────────────────────────────┐
│ Уровень 1: Промпты границ поведения                  │
│ Системные инструкции ИИ явно запрещают:              │
│   ✗ Установка/удаление ПО без запроса                │
│   ✗ Изменение переменных окружения или системных конфигов │
│   ✗ Выполнение деструктивных операций                │
│   ✓ Запросы «проверить/осмотреть» → команды только для чтения │
│   ✓ Найденные проблемы → сначала отчёт, никогда не исправляет сам │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│ Уровень 2: Классификация команд SafetyGuard          │
│ Каждая команда проверяется перед выполнением:        │
│   🔴 заблокировано → Блокируется немедленно, никогда не выполняется │
│      (rm -rf /, chmod 777, форматирование диска и т.д.) │
│   🟡 предупреждение → Всплывающее предупреждение, требует ввода ПОДТВЕРДИТЬ │
│      (apt install, systemctl stop, изменения брандмауэра) │
│   🔵 информация → Низкорисковое уведомление, выполняется обычно │
│      (curl, wget, ls, cat и т.д.)                    │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│ Уровень 3: Вы — финальный шлюз                       │
│ Вы всегда последняя линия обороны.                    │
│ Команды уровня предупреждения не выполняются без ПОДТВЕРЖДЕНИЯ. │
│ Вы можете прервать, отменить или проверить в любое время. │
└─────────────────────────────────────────────────────┘
```

### 📋 Хартия поведения агента

| Что вы можете попросить | Что сделает ИИ | Чего ИИ не сделает |
|:---|:---|:---|
| Установить ПО | Сгенерирует официальные команды установки и выполнит их | Самостоятельно решит, какую версию устанавливать |
| Проверить безопасность | Выполнит аудитные команды и сообщит о находках | Исправит проблемы без вашего разрешения |
| Настроить среду | Строго последует официальной документации | Изменит системные параметры, о которых вы не просили |
| Читать логи | Отфильтрует и покажет ключевую информацию | Удалит или изменит файлы логов |
| Управлять сервисами | Запустит/остановит указанные вами сервисы | Запустит другие сервисы, которые вы не упомянули |
| Запускать рабочие процессы | Автоматически выполнит предопределённые шаги | Пропустит критические шаги или изменит процесс |

**Кратко: ИИ — ваш помощник, а не начальник. Он делает то, о чём вы попросите. Не более.**

**Разбор потока:**

1. Пользователь вводит «установи openclaw» → агент извлекает операцию (установить) и платформу (linux)
2. База знаний сопоставляет `openclaw` для `linux-debian` (строгий режим), внедряя официальные команды установки
3. Агент точно следует базе знаний: устанавливает Node.js 22, затем `npm install -g openclaw`
4. Проверка после установки: запускает `openclaw --version`, чтобы подтвердить успех

> 💡 База знаний поддерживает сопоставление по платформам (`linux-debian` vs `linux-rhel` дают разные команды пакетных менеджеров), с удалёнными обновлениями в один клик
