# AI Terminal v2.0.0 体验优化方案

> 基于产品设计评审的 P0-P2 级优化项，按优先级分阶段执行。

---

## 一、P0 级：立刻改（阻塞体验问题）

### P0-1：AI 面板改为可完全收起

**现状问题**
- AI 面板默认占用 40% 屏幕空间，终端区域被严重挤压
- 用户无法完全关闭 AI 面板，只能调整比例（15%-70%）
- 对于纯终端用户，AI 面板是持续存在的视觉干扰

**优化目标**
- AI 面板支持「完全收起/展开」两种状态，而非仅调整比例
- 默认状态：收起（最大化终端工作区域）
- 提供显式的展开/收起按钮，状态持久化到本地存储

**实现细节**

1. **状态管理**
   - 在 `TerminalPage` 添加 `bool _isAIPanelVisible` 状态
   - 持久化到 `HiveInit.settingsBox.put('aiPanelVisible', bool)`
   - 初始化时读取该值，默认 `false`（收起）

2. **UI 改造**
   - 收起状态：终端区域占 100%，在顶部栏右侧添加 AI 展开按钮（图标：`Icons.auto_fix_high`）
   - 展开状态：保持现有分栏布局，但在分割条上添加收起按钮
   - 展开/收起切换时添加 250ms 的宽度动画（`AnimatedContainer`）

3. **按钮位置**
   - 收起状态下：顶部栏最右侧，连接状态徽章左侧
   - 展开状态下：AI 分割条顶部，与布局切换按钮并列

4. **移动端适配**
   - 移动端 AI 面板以 BottomSheet 形式弹出，而非固定分栏
   - 点击 AI 按钮弹出 BottomSheet，点击外部或拖拽关闭

**验收标准**
- [ ] AI 面板可以完全收起，终端区域占满全屏
- [ ] 收起/展开状态切换有平滑动画
- [ ] 状态持久化，重启后保持上次设置
- [ ] 移动端 AI 以 BottomSheet 形式工作

### P0-2：删除启动页 2 秒强制延迟

**现状问题**
- `main.dart` 中硬编码 `Future.delayed(const Duration(seconds: 2))`
- 所有初始化（Hive、供应商配置、知识库）在后台已完成
- 用户被迫等待 2 秒看 Logo，无实际加载需求

**优化目标**
- 初始化完成后立刻进入主界面
- 保留启动页作为过渡，但仅在真正需要加载时显示
- 加载时间 < 500ms 时不显示启动页

**实现细节**

1. **启动逻辑重构**
   ```dart
   // 原逻辑：先显示 SplashScreen，2秒后跳转
   // 新逻辑：并行初始化，完成后直接显示 App
   
   void main() async {
     WidgetsFlutterBinding.ensureInitialized();
     
     final stopwatch = Stopwatch()..start();
     await HiveInit.init();
     await ProviderConfigService.init();
     // 知识库异步初始化，不阻塞
     KnowledgeService().init().catchError((e) => debugPrint(...));
     
     final elapsed = stopwatch.elapsedMilliseconds;
     final showSplash = elapsed < 500; // 初始化快则不显示启动页
     
     runApp(ProviderScope(
       child: showSplash ? SplashScreen(delayMs: 500 - elapsed) : const AISTerminalApp(),
     ));
   }
   ```

2. **启动页改造**
   - `SplashScreen` 接收 `delayMs` 参数，作为剩余显示时间
   - 最小显示时间 500ms（避免闪屏）
   - 添加淡出动画（FadeTransition），持续 200ms

3. **加载状态指示**
   - 如果初始化超过 500ms，在启动页显示加载进度文本
   - 文本内容："正在初始化..." → "正在加载配置..." → "准备就绪"

**验收标准**
- [ ] 初始化 < 500ms 时，用户几乎感知不到启动页
- [ ] 初始化 > 500ms 时，启动页显示实际加载状态
- [ ] 启动页有淡出动画，不突兀

### P0-3：字体大小调整

**现状问题**
- 标题 `fTitle = 16.0` —— 在桌面端过小
- 正文 `fBody = 13.0` —— 低于现代应用标准（14-16px）
- 终端字体默认 13px —— 长时间阅读疲劳
- 小字 `fSmall = 11.0` —— 接近不可读边界

**优化目标**
- 建立清晰的字体层级，符合桌面端阅读习惯
- 终端字体适当放大，提升长时间工作舒适度

**实现细节**

1. **字体常量调整**
   ```dart
   // 字体系统
   const double fDisplay = 24.0;     // 页面大标题（新增）
   const double fTitle = 18.0;       // 卡片/区域标题（原 16→18）
   const double fBody = 14.0;        // 正文（原 13→14）
   const double fSmall = 12.0;       // 辅助文字（原 11→12）
   const double fMono = 14.0;        // 终端等宽字体（原 12→14）
   const double fMicro = 11.0;       // 标签/徽章（原 10→11）
   ```

2. **终端字体默认调整**
   - `TerminalPage` 中 `_terminalFontSize` 默认值从 13 改为 14
   - 字号选择范围调整为 [10, 11, 12, 13, 14, 15, 16, 18, 20, 22, 24]
   - 保存到 Hive 的键从 `terminalFontSize` 迁移，旧值读取后 +1 补偿

3. **组件级适配**
   - `HostCard` 中服务器名称从 `fontSize: 16` 改为 `fTitle (18)`
   - `HostCard` 中 IP/用户名从 `fontSize: 13` 改为 `fBody (14)`
   - `ChatPage` 消息气泡文字从默认改为 `fBody`
   - `SettingsPage` 列表项标题从默认改为 `fBody`

4. **行高调整**
   - 正文行高从默认改为 1.5（提升阅读舒适度）
   - 终端行高保持 1.4（代码紧凑性）

**验收标准**
- [ ] 所有页面文字清晰可读，无过小字体
- [ ] 终端默认字号 14px，用户可感知到变化
- [ ] 字体层级清晰：Display > Title > Body > Small > Micro

## 二、P1 级：本周改（核心体验优化）

### P1-4：服务器列表重设计

**现状问题**
- 卡片高度固定 72px，信息密度低
- 空状态只有图标和文字，无引导
- 分组默认全部展开，失去分组意义
- 无连接历史、最后在线时间等信息
- 搜索只有简单 contains，无高亮

**优化目标**
- 提升信息密度，一屏展示更多服务器
- 增加连接历史、状态时间等元信息
- 优化空状态，增加情感化引导
- 搜索支持结果高亮

**实现细节**

1. **卡片高度与布局**
   - 高度从 72px 增加到 88px
   - 左侧状态点改为 12px 圆形，增加发光效果（在线时）
   - 增加服务器图标/头像区域（40px 圆形，首字母或自定义颜色）
   - 右侧操作区：直接显示快捷操作按钮（连接、SFTP），而非隐藏在下拉菜单

2. **信息展示优化**
   ```
   [状态点] [头像] 服务器名称              [连接按钮] [SFTP按钮]
              username@host:port · 最后连接: 2小时前
              [标签颜色条] [分组标签]
   ```

3. **连接状态增强**
   - `lastConnectedAt` 字段已存在，格式化显示相对时间
   - 新增 `connectionCount` 统计（连接次数）
   - 在线状态增加脉冲动画（`AnimatedContainer` 循环缩放）

4. **分组交互优化**
   - 默认只展开"默认"分组，其他分组收起
   - 记录用户展开/收起状态到 Hive
   - 分组标题增加服务器数量徽章（保持现有）

5. **搜索高亮**
   - 搜索结果中高亮匹配文本（`TextSpan` 实现）
   - 支持按名称、主机、用户名、分组搜索
   - 搜索无结果时显示"未找到匹配服务器" + 清除搜索按钮

6. **空状态重设计**
   - 图标：从 `Icons.storage` 改为动态终端动画（Lottie 或自定义）
   - 标题："还没有服务器"
   - 副标题："添加第一台服务器，开启 AI 辅助运维之旅"
   - CTA：大号主按钮"添加服务器" + 次要按钮"了解功能"

7. **快捷操作**
   - 左滑卡片显示快捷操作（编辑/删除/复制）
   - 长按卡片弹出操作菜单
   - 桌面端右键菜单

**验收标准**
- [ ] 一屏至少展示 6-8 个服务器（桌面端）
- [ ] 卡片包含最后连接时间
- [ ] 搜索高亮匹配文本
- [ ] 空状态有情感化引导和 CTA
- [ ] 分组状态持久化

### P1-5：设置页信息架构重组

**现状问题**
- 按技术模块划分，而非用户心智模型
- "内置系统提示词"和"自定义 AI 提示词"分开
- "Agent 设置"只有一个参数
- "更新知识库"位置不合理

**优化目标**
- 按用户任务场景重组：连接、AI、体验、关于
- 合并相关功能，减少入口数量
- 高频功能前置，低频功能折叠

**实现细节**

1. **新信息架构**
   ```
   设置页
   ├── 连接管理（原服务器管理）
   │   ├── 服务器列表
   │   └── 快捷命令
   ├── AI 与智能
   │   ├── AI 模型配置
   │   ├── Agent 设置（合并提示词设置）
   │   │   ├── 最大执行步骤数
   │   │   ├── 内置系统提示词（折叠）
   │   │   └── 自定义提示词（折叠）
   │   └── 命令手册知识库
   ├── 外观与体验
   │   ├── 主题模式
   │   ├── 终端字体大小
   │   └── 终端配色方案（新增）
   └── 关于
       ├── 版本信息
       ├── 检查更新
       └── 安全设置（移入）
   ```

2. **Agent 设置合并**
   - 新建 `AgentSettingsPage`，整合：
     - 最大执行步骤数（Slider + 数字输入）
     - 内置提示词（折叠面板，默认收起）
     - 自定义提示词（折叠面板，默认收起）
   - 提供"重置为默认"按钮

3. **安全设置降级**
   - 从一级入口移入"关于"分组底部
   - 添加生物识别状态指示

4. **视觉层级**
   - 分组标题使用 `fTitle` 字号 + `cPrimary` 颜色
   - 分组间距增加到 24px
   - 列表项增加 hover 效果（桌面端）

**验收标准**
- [ ] 设置页按新架构重组
- [ ] Agent 相关设置合并到一个页面
- [ ] 安全设置移入关于分组
- [ ] 每个分组有清晰的视觉分隔

### P1-6：AI 模型配置简化

**现状问题**
- 展开后显示所有字段（API Key、Base URL、Temperature、Max Tokens）
- Temperature 用 Slider，普通用户不理解
- 测试连接按钮在对话框底部，需要滚动
- 模型列表用 `ExpansionTile` 展开，操作繁琐

**优化目标**
- 默认只显示核心字段，高级参数折叠
- 测试连接更醒目
- 模型卡片操作更便捷

**实现细节**

1. **添加/编辑对话框简化**
   - 默认显示字段：
     - 提供商（Dropdown）
     - 名称（Input）
     - 模型名称（Input + 快捷芯片）
     - API Key（Input）
     - 设为默认（Switch）
   - 高级设置（折叠面板）：
     - Base URL（Input，默认从提供商填充）
     - Temperature（Slider，范围 0-1，默认 0.3）
     - Max Tokens（Input，默认 4096）

2. **测试连接优化**
   - 测试按钮放在 API Key 字段右侧，紧邻输入框
   - 测试结果用图标 + 简短文本显示在按钮旁边
   - 测试成功后才允许保存

3. **模型卡片优化**
   - 卡片不再用 `ExpansionTile`，改为固定高度卡片
   - 卡片内直接显示：名称、提供商、模型、默认标记
   - 操作按钮外露：编辑（图标）、删除（图标）、设为默认（星形图标）
   - 点击卡片非按钮区域展开详情（可选）

4. **提供商选择优化**
   - Dropdown 中显示提供商图标 + 名称 + 简介
   - 选择后自动填充 Base URL 和推荐模型
   - 增加"自定义"选项，允许手动输入所有字段

**验收标准**
- [ ] 默认只显示 5 个核心字段
- [ ] 高级参数在折叠面板中
- [ ] 测试按钮紧邻 API Key 字段
- [ ] 模型卡片操作按钮外露

   ```dart
   const double rXSmall = 4.0;   // 标签、徽章
   ```

- [ ] 阴影在悬浮态正确应用
- [ ] 新增紫罗兰辅助色用于 AI 元素

### P2-8：组件库建设

**现状问题**
- 消息气泡在 `terminal_page.dart` 和 `chat_page.dart` 中实现不一致
- 代码块渲染逻辑内联在页面中
- 无统一的空状态组件

**优化目标**
- 提取统一的消息气泡组件
- 提取代码块组件
- 创建空状态组件

   ```dart
     final bool isLoading;
     final bool isStreaming;
     final VoidCallback? onCopy;
     final VoidCallback? onExecute;
     final List<String>? options; // :::choose::: 选项
     final Function(String)? onOptionSelected;
     
     // 统一样式：
     // - 用户消息：右对齐，主色背景，圆角不对称
     // - AI 消息：左对齐，卡片背景，圆角不对称
     // - 加载状态：脉冲动画指示器
     // - 选项按钮：横排 Wrap 布局
   }
   ```

2. **代码块组件 `CodeBlockWidget`**
   ```dart
   class CodeBlockWidget extends StatelessWidget {
     final String code;
     final String? language;
     final bool executable; // 是否显示执行按钮
     final VoidCallback? onExecute;
     final VoidCallback? onCopy;
   ```

3. **空状态组件 `EmptyStateWidget`**
   ```dart
   class EmptyStateWidget extends StatelessWidget {
     final IconData icon;
     final String title;
     final String? subtitle;
     final String? actionLabel;
     final VoidCallback? onAction;
     final Widget? customIllustration; // 支持自定义动画/插画
     
     // 统一样式：
     // - 图标：64px，主色，轻微浮动动画
     // - 标题：fTitle，textMain
     // - 副标题：fBody，textSub
     // - 按钮：主按钮样式
   }
   ```

4. **组件替换计划**
   - `terminal_page.dart`：`_buildMessageBubble` → `UnifiedMessageBubble`
   - `terminal_page.dart`：`_buildCodeBlockWithExecuteButton` → `CodeBlockWidget`
   - `chat_page.dart`：`MessageBubble` → `UnifiedMessageBubble`
   - `host_list_page.dart`：`_buildEmptyState` → `EmptyStateWidget`
   - `ai_models_page.dart`：`_buildEmptyState` → `EmptyStateWidget`

**验收标准**
- [ ] 所有消息气泡样式统一
- [ ] 代码块组件支持执行/复制/安全指示
- [ ] 空状态组件支持自定义插画
- [ ] 组件在 terminal 和 chat 页面表现一致

### P2-9：空状态情感化设计

**现状问题**
- 所有空状态都是 `Icon` + `Text` + `Button` 三板斧
- 无情感化元素，无品牌个性
- 无引导用户下一步的明确路径

**优化目标**
- 每个空状态有独特的视觉表达
- 引导文案有温度，不机械
- 提供明确的下一步操作

**实现细节**

1. **服务器列表空状态**
   - 视觉：终端光标闪烁动画（`_` 字符闪烁）
