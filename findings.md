# AI Terminal UI/UX 调研发现

## 现有产品问题

### 1. 视觉层次混乱
- 终端区域为纯黑 (#000000)，与外围深色 UI (#111118) 形成强烈割裂感，不像一个整体应用。
- AI 面板与终端之间的橙色分隔线过于突兀，像临时拼接。
- 顶部栏图标按钮过小且密，没有文字标签，对新手不友好。

### 2. 品牌感弱
- 当前 LOGO 为复杂的 AI 大脑 + 终端窗口组合，细节过多，在小尺寸下失真，且带有生成水印。
- App 图标是彩色描边插画风格，与深色专业开发者工具气质不符。
- 应用名称 "ai_terminal" 直接作为标题显示，缺乏品牌处理。

### 3. 组件风格不统一
- 命令块顶部的红黄绿"窗口按钮"是过时的拟物隐喻，与整体扁平风格冲突。
- 不同页面卡片的圆角、间距、阴影不统一（命令块 8px、卡片 16px、按钮 6px 但样式各异）。
- 结果卡片文字对比度不足，灰色 (#8888A0) 在深色背景上可读性差。

### 4. 信息密度与布局
- 主机列表页分组标题颜色与卡片过于接近，分组边界不清晰。
- AI 输入区底部工具按钮过多且拥挤，模型选择器、Agent 标签、日志按钮、会话按钮堆叠在一起。
- 消息气泡内边距和圆角在不同角色（用户/AI）间不一致。

### 5. 交互反馈不足
- 按钮 hover/pressed 状态不明显。
- 流式输出时的光标指示器较简陋。
- 加载、空状态、错误状态缺乏品牌化的插图或动效。

## 竞品设计语言参考

### Warp (warp.dev)
- **核心理念**：将终端从"字符流"升级为"块结构"（Blocks）—— 每条命令和输出形成一个可交互的卡片。
- **视觉风格**：深色背景略带蓝紫色调，命令块使用 subtle 的分隔而非强边框；AI 面板以侧边栏形式出现。
- **可借鉴**：命令/结果的分块思维、AI 输入的命令行化（Command Palette 风格）、紧凑但不拥挤的工具栏。

### Tabby (tabby.sh)
- **核心理念**：跨平台、可高度定制的传统标签式终端。
- **视觉风格**：简洁的标签栏、左侧连接列表、设置页使用卡片分组。
- **可借鉴**：主机列表的侧边栏布局（桌面端）、连接配置表单的分组组织。

### Linear / Vercel
- **核心理念**：开发者工具的"精致极简"—— 大量留白、微妙的边框、克制的颜色、清晰的字体层级。
- **视觉风格**：深色背景使用深蓝灰（如 #0F1117 / #111827），而非纯黑；强调色仅用于关键操作；圆角统一（多为 6-8px）。
- **可借鉴**：颜色系统、间距系统、按钮/输入框的聚焦态、空状态设计。

### Claude / ChatGPT Desktop
- **核心理念**：对话式 AI 界面，强调"人在回路"和渐进式信息披露。
- **视觉风格**：消息气泡清晰区分用户与 AI；AI 回复中的代码块有独立样式和复制按钮；输入框始终可见。
- **可借鉴**：AI 消息的结构化展示、命令与解释的分离、思考/执行状态的清晰提示。

## 设计趋势总结

1. **深色主题向"深灰"靠拢**：纯黑背景正在减少，取而代之的是深蓝灰/炭灰，减少眼睛疲劳并提升 UI 元素区分度。
2. **Bordered Cards 回归**：细边框卡片（1px，低对比度边框）比阴影更能营造秩序感，尤其适合深色模式。
3. **Monospace 仅用于代码/终端**：界面文字使用无衬线系统字体，代码区域保留等宽字体。
4. **AI 状态可视化**：使用脉冲、微光、渐变等效果表现 AI 思考，而非简单的转圈。
5. **Command Palette 化**：AI 输入框被设计为命令面板风格，支持 / 命令、快捷键、上下文感知。
6. **品牌色克制**：主色只用于 10-15% 的关键元素，大量使用中性和语义色。

## 决策记录

- 保留深色主题为默认，但将背景从纯黑调整为统一的深蓝灰系，让终端与 UI 更融合。
- 命令块放弃"红黄绿窗口按钮"，改用简洁的状态标签 + 左侧色条。
- 终端和 AI 面板之间取消醒目的分隔线，改用细微边框或背景色差异区分。
- 顶部栏重构为"品牌区 + 标签页 + 全局操作"三段式，增加文字标签和视觉节奏。
- 设计新 LOGO：以终端提示符 ">_" 和 AI  sparkle/节点为灵感的几何图形，而非具象大脑。
